  <action>
**1. Install Husky and lint-staged:**

```bash
npm install --save-dev husky lint-staged
```

**2. Initialize Husky:**

```bash
npx husky init
```

This creates `.husky/` directory and adds a `prepare` script to package.json.

**3. Configure the pre-commit hook:**

Replace the default `.husky/pre-commit` content with:

```bash
npx lint-staged
```

**4. Create `.lintstagedrc.json`** at the project root:

```json
{
  "*.{ts,tsx}": [
    "eslint --fix --max-warnings 0"
  ]
}
```

This runs ESLint with auto-fix on all staged TypeScript files. The `--max-warnings 0` ensures warnings are treated as errors in the pre-commit context.

**Note on type-checking:** Do NOT add `tsc --noEmit` to lint-staged because tsc needs the full project context (it cannot type-check individual files). Instead, the type-check runs as part of `npm run build` (which is `tsc -b && vite build`) and will be enforced in the CI workflow (Plan 02). Running tsc in lint-staged would type-check the entire project on every commit regardless of what files changed, which is slow and redundant with CI.

**5. Verify the prepare script was added to package.json:**

After `npx husky init`, package.json should have:
```json
{
  "scripts": {
    "prepare": "husky"
  }
}
```

If not present, add it manually.

**6. Test the hook works:**

Stage a test change and verify lint-staged runs. Then unstage it.
