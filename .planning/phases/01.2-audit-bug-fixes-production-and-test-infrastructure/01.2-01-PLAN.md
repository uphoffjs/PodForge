  <action>
Two fixes in `cypress/support/commands.js`:

**Fix 1 — mockEventPage PostgREST .single() response (Finding 3):**

The current `mockEventPage` command returns `body: [event]` (an array) with no content-type header. But `useEvent.ts` uses `.single()` which sends `Accept: application/vnd.pgrst.object+json`. PostgREST returns a plain object (not array) with that content-type.

Change the event intercept (lines 30-33) from:
```js
cy.intercept('GET', '**/rest/v1/events*', {
  statusCode: 200,
  body: [event],
}).as('getEvent')
```

To:
```js
cy.intercept('GET', '**/rest/v1/events*', {
  statusCode: 200,
  body: event,
  headers: {
    'content-type': 'application/vnd.pgrst.object+json; charset=utf-8',
  },
}).as('getEvent')
```

This matches the pattern already used correctly in `event-creation.cy.js` lines 37-43.

**Fix 2 — createRealEvent RPC parameter names (Finding 2):**

Change line 73 from:
```js
body: { event_name: name, admin_passphrase: passphrase },
```

To:
```js
body: { p_name: name, p_passphrase: passphrase },
```

These parameter names match the `create_event` RPC function signature in `supabase/migrations/00001_initial_schema.sql` and the production code in `src/hooks/useCreateEvent.ts`.

**Also update self-drop.cy.js's setupJoinedPlayer if it uses its own event intercept with the array pattern.** Check: self-drop.cy.js line 40-43 returns `body: [event]` — this also needs to be fixed to return `body: event` with the correct content-type header, same as mockEventPage. Apply the same fix pattern.

Fix the localStorage player identity format in `visual-regression.cy.js` (Finding 1). Two changes:

**Change 1 -- "Event Page (with players)" describe block (~line 78):**

Replace the JSON.stringify call with a plain UUID string. Change from:
```js
win.localStorage.setItem(
  `podforge_player_${event.id}`,
  JSON.stringify({ id: 'player-1', name: 'Alice' })
)
```
To:
```js
win.localStorage.setItem(
  `podforge_player_${event.id}`,
  'player-1'
)
```

This matches the correct pattern in `self-drop.cy.js` line 53. The player-1 ID exists in the mocked players array, so the validation effect will NOT clear it.

**Change 2 -- "Event Page (empty state)" describe block (~line 104):**

Remove the `onBeforeLoad` localStorage setup entirely for this test. Instead, after the page loads with the join form visible (no stored identity), click the skip button to dismiss it, then capture the snapshot. The updated test flow:
1. `cy.mockEventPage(event, [])` -- mocks event with empty players
2. `cy.visit(...)` -- page loads, join form renders (no stored identity)
3. `cy.getByTestId('join-skip-btn').click()` -- dismisses join form
4. `waitForContent('[data-testid="player-list-empty"]')` -- wait for empty state visible
5. Capture snapshot

This is necessary because storing a fake player ID (e.g., `'player-solo'`) would be cleared by the validation effect since no matching player exists in the empty array, causing the join form to reappear. The skip-button approach is the correct way to show the empty player list without the join form overlay.
