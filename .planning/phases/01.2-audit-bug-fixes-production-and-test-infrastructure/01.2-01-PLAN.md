Two fixes in `cypress/support/commands.js`:

**Fix 1 — mockEventPage PostgREST .single() response (Finding 3):**

The current `mockEventPage` command returns `body: [event]` (an array) with no content-type header. But `useEvent.ts` uses `.single()` which sends `Accept: application/vnd.pgrst.object+json`. PostgREST returns a plain object (not array) with that content-type.

Change the event intercept (lines 30-33) from:
```js
cy.intercept('GET', '**/rest/v1/events*', {
  statusCode: 200,
  body: [event],
}).as('getEvent')
```

To:
```js
cy.intercept('GET', '**/rest/v1/events*', {
  statusCode: 200,
  body: event,
  headers: {
    'content-type': 'application/vnd.pgrst.object+json; charset=utf-8',
  },
}).as('getEvent')
```

This matches the pattern already used correctly in `event-creation.cy.js` lines 37-43.

**Fix 2 — createRealEvent RPC parameter names (Finding 2):**

Change line 73 from:
```js
body: { event_name: name, admin_passphrase: passphrase },
```

To:
```js
body: { p_name: name, p_passphrase: passphrase },
```

These parameter names match the `create_event` RPC function signature in `supabase/migrations/00001_initial_schema.sql` and the production code in `src/hooks/useCreateEvent.ts`.

**Also update self-drop.cy.js's setupJoinedPlayer if it uses its own event intercept with the array pattern.** Check: self-drop.cy.js line 40-43 returns `body: [event]` — this also needs to be fixed to return `body: event` with the correct content-type header, same as mockEventPage. Apply the same fix pattern.

<task type="auto">
  <name>Task 2: Fix localStorage format in visual-regression.cy.js</name>
  <files>cypress/e2e/visual-regression.cy.js</files>
  <action>
Fix the localStorage player identity format in `visual-regression.cy.js` (Finding 1).

The current code at lines 78-81 stores a JSON object:
```js
win.localStorage.setItem(
  `podforge_player_${event.id}`,
  JSON.stringify({ id: 'player-1', name: 'Alice' })
)
```

But `src/lib/player-identity.ts` `getStoredPlayerId()` returns raw `localStorage.getItem()` — a plain string. The EventPage validation effect compares this stored value against `players.some(p => p.id === currentPlayerId)`. The JSON stringified object `'{"id":"player-1","name":"Alice"}'` will never match `'player-1'`, causing the identity to be cleared and the join form to render instead of the player view.

**Fix all occurrences.** There are two places that set localStorage in this file:

1. "Event Page (with players)" describe block (~line 78): Change to store plain UUID string `'player-1'`
2. "Event Page (empty state)" describe block (~line 104): Change to store plain UUID string `'player-solo'`

For each, change from:
```js
win.localStorage.setItem(
  `podforge_player_${event.id}`,
  JSON.stringify({ id: 'player-1', name: 'Alice' })
)
```

To:
```js
win.localStorage.setItem(
  `podforge_player_${event.id}`,
  'player-1'
)
```

And similarly for the `player-solo` case. This matches the correct pattern used in `self-drop.cy.js` line 53: `win.localStorage.setItem(storageKey, playerId)`.

**Important:** After changing the empty-state test, the `player-solo` ID does NOT exist in the empty players array `[]`. The validation effect will clear it. Since we want to show the empty player list (not the join form), the test also needs to skip join. Two options:
- Add `player-solo` to the empty players array (but that defeats "empty state")
- Use `skippedJoin` by setting it via the page's skip button after load

The simplest fix: The empty state test should set `skippedJoin` state. Since this is test setup and we can't directly set React state, use `onBeforeLoad` to set localStorage AND after page load click the skip button. OR: better, just don't set a player identity at all, let the join form render, click the skip button, and then capture the empty state. BUT the test needs the empty player list visible without the join form overlay.

Best approach: Keep setting localStorage with a player ID, but add that player to the players list as a "viewer" so validation doesn't clear it. No — that defeats empty state.

Actually re-read the test: the empty state test uses `waitForContent('[data-testid="player-list-empty"]')` which is the "No players yet" message. The join form and the player list both render simultaneously (join form overlays). So if identity is cleared and join form shows, the player-list-empty is still in the DOM (behind the overlay). But for the visual snapshot, we want NO join form visible.

Simplest correct fix: Use the same `onBeforeLoad` pattern from self-drop.cy.js. Set localStorage before visit so React reads it on mount. But validation effect runs after players fetch and will clear it since the player isn't in the list.

Best fix for the empty-state visual test: Instead of setting a fake player identity, use the `skippedJoin` mechanism. After `cy.mockEventPage(event, [])`, click the skip button to hide the join form, then capture the snapshot. Change the empty-state test to:
1. Remove the localStorage setup entirely
2. After `cy.mockEventPage(event, [])`, the join form renders (no identity)
3. Click `cy.getByTestId('join-skip-btn').click()` to dismiss the join form
4. Then `waitForContent('[data-testid="player-list-empty"]')` and capture snapshot

This is the cleanest approach. The "with players" test correctly stores a plain UUID that exists in the players list, so validation won't clear it.
