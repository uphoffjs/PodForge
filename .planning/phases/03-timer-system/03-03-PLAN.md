**1. Create `src/hooks/useTimerNotification.ts`:**

```typescript
import { useEffect, useRef, useCallback, useState } from 'react'
import type { RoundTimer } from '@/types/database'
import type { CountdownState } from '@/hooks/useCountdown'

interface UseTimerNotificationReturn {
  /** Whether the browser supports notifications */
  isSupported: boolean
  /** Current permission state: 'default' | 'granted' | 'denied' */
  permission: NotificationPermission | 'unsupported'
  /** Request notification permission (call on user interaction, NOT on mount) */
  requestPermission: () => Promise<void>
}

export function useTimerNotification(
  timer: RoundTimer | null,
  countdown: CountdownState | null
): UseTimerNotificationReturn
```

**Logic:**

**Permission management:**
- Check `'Notification' in window` for support detection. iOS PWA may have window.Notification but requestPermission may fail — wrap in try/catch.
- `permission` state initialized from `Notification.permission` (or 'unsupported')
- `requestPermission`: calls `Notification.requestPermission()`, updates state. Wrapped in try/catch for iOS PWA where this may throw.
- Do NOT request permission on mount. Permission is requested explicitly when the admin starts a timer (triggered by the caller).

**Notification firing:**
- Use `useRef<string | null>(null)` to track `lastNotifiedTimerId` — prevents firing multiple times for the same timer
- Use `useEffect` watching countdown:
  - If countdown is null, or countdown.isPaused, or countdown.isCancelled → do nothing
  - If countdown.remainingSeconds <= 0 AND countdown.isOvertime AND timer is not null:
    - If `lastNotifiedTimerId.current !== timer.id`:
      - Set `lastNotifiedTimerId.current = timer.id`
      - If permission === 'granted': fire `new Notification("Time's Up!", { body: "Round timer has expired", icon: '/favicon.ico', tag: 'timer-expired' })`
      - The `tag: 'timer-expired'` prevents duplicate notifications if effect runs multiple times
  - If timer changes (new timer.id), reset lastNotifiedTimerId to allow notification for the new timer

**iOS PWA handling:**
- `new Notification()` may throw in iOS PWA context. Wrap in try/catch, silently fail.
- Permission request may throw — catch and set permission to 'denied'.
- isSupported returns false if Notification API is not available
