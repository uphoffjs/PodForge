    ## 2. Create E2E test spec: cypress/e2e/pods-of-3.cy.js

    Follow the `generate-round.cy.js` pattern exactly for setup. Create a `setupAdminEventPage` helper with the same intercepts.

    **Test cases:**

    a. **'shows Allow pods of 3 checkbox in admin controls'**
       - Setup with 7 players
       - Assert `pods-of-3-checkbox` is visible and unchecked

    b. **'generates round with pods of 3 when toggle is enabled'**
       - Setup with 7 players
       - Check the `pods-of-3-checkbox`
       - Mock generate_round RPC
       - Override rounds/pods intercepts to return pods-of-3 fixture after generation
       - Click Generate Round
       - Wait for RPC
       - Assert round display appears
       - Assert pod cards appear: one pod with 4 players, one pod with 3 players
       - Verify the 3-player pod shows exactly 3 player names and seat labels (1st, 2nd, 3rd)

    c. **'generates round WITHOUT pods of 3 when toggle is off (default)'**
       - Setup with 9 players (original behavior: 2 pods of 4 + 1 bye)
       - Do NOT check the checkbox
       - Mock generate_round RPC
       - Override rounds/pods to return standard pods.json fixture
       - Click Generate Round
       - Wait for RPC
       - Assert bye pod appears (existing behavior preserved)

    d. **'shows warning toast for 5 players with toggle enabled'**
       - Setup with exactly 5 players
       - Check the `pods-of-3-checkbox`
       - Mock generate_round RPC
       - Override rounds/pods after generation
       - Click Generate Round
       - Assert Sonner warning toast appears containing '5 players'
       - Assert round still generates (1 pod of 4 + 1 bye)

    e. **'checkbox resets after successful generation'**
       - Setup with 7 players
       - Check the `pods-of-3-checkbox`

       - Mock generate_round RPC
       - Override rounds/pods after generation
       - Click Generate Round
       - Wait for RPC
       - Assert `pods-of-3-checkbox` is now unchecked

    f. **'checkbox is hidden when event is ended'**
       - Setup with event status 'ended' and `isEventEnded` condition
       - Assert `pods-of-3-checkbox` does not exist

    - Use `cy.getByTestId()` custom command for all selectors (not `cy.get('.class')`)

    - Use `.should()` chains that auto-retry
    - Player data arrays should be defined at the top of the describe block
