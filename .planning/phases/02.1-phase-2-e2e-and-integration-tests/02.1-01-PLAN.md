**cypress/e2e/admin-passphrase.cy.js** — Tests for admin authentication gating. This spec tests whether the admin passphrase in sessionStorage correctly controls visibility of admin UI elements.

Because `isAdmin === !!passphrase`, AdminControls and AdminPlayerActions only render when passphrase is already set. The passphrase modal (`AdminPassphraseModal`) is a defensive component for edge cases and cannot be triggered through normal E2E flow. The modal is already covered by unit tests in `AdminPassphraseModal.test.tsx`. This spec instead tests the observable admin authentication behavior end-to-end.

Create a `setupEventPage(options)` helper that accepts `{ asAdmin: boolean }`:

Important intercept note for tests with existing rounds: The intercepts need to handle both useRounds (returns array) and useCurrentRound (returns single object). Both query `**/rest/v1/rounds*`. The simplest approach:
- Return the rounds array for ALL `**/rest/v1/rounds*` requests. supabase-js handles `.maybeSingle()` client-side based on the `Accept` header and response format.
- OR: intercept specifically based on query parameters. useCurrentRound adds `limit=1` to the URL.

Simplest working approach: Intercept `**/rest/v1/rounds*` and check if the request URL contains `limit=1` to differentiate. If it does, return a single-object response for currentRound. Otherwise, return array for useRounds.

Actually, the most reliable approach for mocking: Use `cy.intercept` with a route handler function that inspects the request URL:
```js
cy.intercept('GET', '**/rest/v1/rounds*', (req) => {
  if (req.url.includes('limit=1')) {
    req.reply({
      statusCode: 200,
      body: round1Object,
      headers: { 'content-type': 'application/vnd.pgrst.object+json; charset=utf-8' }
    })
  } else {
    req.reply({ statusCode: 200, body: roundsArray })
  }
}).as('getRounds')
```

For pods, there are two hooks: usePods (by roundId) and useAllRoundsPods (by multiple roundIds using .in()). Both query `**/rest/v1/pods*`. The usePods hook has `round_id=eq.{id}` and useAllRoundsPods has `round_id=in.({ids})`. Differentiate by checking for `in.` vs `eq.` in the URL, or just return the same pods array for all pod queries.
