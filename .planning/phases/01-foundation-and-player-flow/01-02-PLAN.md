  <action>
    **src/hooks/useAdminAuth.ts** -- Admin passphrase session management:
    - `useAdminAuth(eventId: string)` hook returns:
      - `isAdmin: boolean` -- whether sessionStorage has a passphrase for this event
      - `passphrase: string | null` -- the stored passphrase (for passing to RPC calls)
      - `setPassphrase(passphrase: string): void` -- stores in sessionStorage
      - `clearPassphrase(): void` -- removes from sessionStorage
    - Storage key format: `pod_pairer_admin_{eventId}`
    - Use `useState` initialized from `sessionStorage.getItem()` so React re-renders when passphrase changes
    - This hook will be used in later phases for admin-gated actions (generate round, remove player, etc.)

    **src/hooks/useCreateEvent.ts** -- React Query mutation:
    - `useCreateEvent()` hook uses `useMutation` from `@tanstack/react-query`
    - `mutationFn` accepts `{ name: string; passphrase: string }` and calls `supabase.rpc('create_event', { p_name: name, p_passphrase: passphrase })`
    - The RPC returns the new event UUID
    - `onSuccess`: no side effects here -- the component handles navigation and passphrase storage
    - `onError`: show toast via sonner: "Failed to create event. Please try again."
    - Return type should include the event ID from the mutation result

    **IMPORTANT:** The `create_event` RPC returns a UUID directly (not wrapped in an object). The Supabase JS client returns it as `{ data: string, error: ... }` where `data` is the UUID string.
  </action>
  <verify>
    `npx tsc --noEmit` passes with no type errors.
    Both hooks export correctly and can be imported.
  </verify>
  <done>
