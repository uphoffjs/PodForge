<objective>
Fix AdminControls to fetch pods for ALL rounds (not just the latest) when building opponent history for the pod generation algorithm.

Purpose: Close the PODG-02 verification gap. Currently, AdminControls only passes the latest round's pods to `generatePods`, causing the greedy opponent-avoidance algorithm to ignore opponents from rounds 1..N-2 when generating round N. This undermines the "minimize repeat opponents across rounds" success criterion.

Output: A new `useAllRoundsPods` hook that fetches pods for all event rounds in one query, updated `AdminControls` that passes complete round history to `generatePods`, and tests verifying the fix.
</objective>
