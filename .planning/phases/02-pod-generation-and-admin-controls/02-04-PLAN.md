**1. Create src/components/AdminPlayerActions.tsx:**

Inline action buttons that appear next to player names when admin is viewing. These are small, unobtrusive buttons that don't clutter the player list for non-admin viewers.

Props:
```typescript
interface AdminPlayerActionsProps {
  eventId: string
  playerId: string
  playerName: string
  playerStatus: 'active' | 'dropped'
  passphrase: string | null
  onPassphraseNeeded: () => void
}
```

Behavior:
- For active players: show a small "Remove" button (lucide `UserMinus` icon, text-error/60 hover:text-error).
- For dropped players: show a small "Reactivate" button (lucide `UserPlus` icon, text-accent/60 hover:text-accent).
- On click:
  1. If passphrase is null, call onPassphraseNeeded().
  2. Otherwise, show ConfirmDialog: "Remove {name}?" / "Reactivate {name}?" with appropriate messaging.
  3. On confirm, call useRemovePlayer or useReactivatePlayer mutation.
  4. Toast on success: "Removed {name}" or "Reactivated {name}".
  5. Toast on error (invalid passphrase, etc.).
- State management: internal `showConfirm` state for the confirmation dialog.
- data-testid: `admin-remove-player-{playerId}`, `admin-reactivate-player-{playerId}`.
