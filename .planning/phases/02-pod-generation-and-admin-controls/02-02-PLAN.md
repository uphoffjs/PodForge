  <behavior>
The `generatePods` function:

**Input:**
```typescript
interface PlayerInfo {
  id: string
  name: string
}

interface RoundHistory {
  pods: { playerIds: string[], isBye: boolean }[]
}

function generatePods(
  activePlayers: PlayerInfo[],
  previousRounds: RoundHistory[]
): PodAssignmentResult
```

**Output:**
```typescript
interface PodAssignment {
  pod_number: number
  is_bye: boolean
  players: { player_id: string, seat_number: number | null }[]
}

interface PodAssignmentResult {
  assignments: PodAssignment[]
  warnings: string[]  // e.g., "High bye count: 3 of 7 players sitting out"
}
```

**Algorithm (greedy with opponent history matrix):**

1. **Validation:** If activePlayers.length < 4, throw Error('Fewer than 4 active players').

2. **Build opponent history matrix:** From previousRounds, create a Map<string, Map<string, number>> where matrix[playerA][playerB] = number of times they were in the same pod.

3. **Build bye count map:** From previousRounds, count how many times each player was in a bye pod. Players not in history have 0 byes.

4. **Select bye players (if needed):**
   - Calculate numByes = activePlayers.length % 4.
   - If numByes > 0, select bye players: sort by fewest byes (ascending), break ties randomly (shuffle within same bye count, take first N).
   - If numByes >= 3 (i.e., 3 of 7 or similar), add warning: `"High bye count: ${numByes} of ${activePlayers.length} players sitting out"`.

5. **Assign remaining players to pods (greedy):**
   - Pool = activePlayers minus bye players.
   - Number of pods = pool.length / 4.
   - Initialize empty pods.
   - For each pod slot, greedily pick the player from the pool who has the lowest total "opponent score" against players already in that pod. Opponent score = sum of matrix[candidate][existingPodMember] for all existing members.
   - Specifically: fill pods one at a time. For each pod, pick the first player randomly from the pool. Then for positions 2-4, pick the player from the remaining pool with the lowest opponent overlap with players already placed in this pod.

6. **Assign seat numbers:** For each non-bye pod, shuffle seat numbers 1-4 among the 4 players. Bye pod players get seat_number = null.

7. **Build result:** Create PodAssignment array with pod_number (1-indexed), is_bye flag, and player entries.

**Test cases (RED phase):**

- 4 players, 0 rounds: 1 pod, all 4 players, seats 1-4 assigned
- 5 players, 0 rounds: 1 pod of 4 + 1 bye player
- 8 players, 0 rounds: 2 pods of 4, no byes
- 12 players, 0 rounds: 3 pods of 4
- 7 players, 0 rounds: 1 pod of 4 + 3 byes, warning generated
- 6 players, 0 rounds: 1 pod of 4 + 2 byes
- 3 players: throws 'Fewer than 4 active players'
- 0 players: throws 'Fewer than 4 active players'
- 8 players, 1 prior round: pods minimize repeats (verify no player shares a pod with more prior opponents than necessary)
- 8 players, 2 prior rounds: opponent avoidance stacks across rounds
- Bye rotation: 5 players across 2 rounds, the player who sat out in round 1 should NOT sit out in round 2 (different bye player selected)
- Seat numbers: all seats 1-4 present in each non-bye pod, bye players have null seats
- Bye players with equal bye counts: selection is non-deterministic (randomized tie-breaking)
- buildOpponentHistory: verify matrix built correctly from round history
