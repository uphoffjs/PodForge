  <action>
Create `.github/workflows/ci.yml` with the following specification:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  quality:
    name: Lint, Type-check & Test
    runs-on: ubuntu-24.04

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Type-check
        run: npx tsc -b

      - name: Unit tests with coverage
        run: npm run test:coverage
```

**Design decisions:**
- **Single job with sequential steps** rather than parallel jobs. The steps are fast (lint ~5s, tsc ~10s, tests ~30s) and a single job avoids the overhead of spinning up multiple runners and installing deps multiple times. Total CI time will be under 2 minutes.
- **Node 22** — matches the LTS version. The project specifies 18+ in DEPLOYMENT.md but modern LTS (22) is preferred for CI.
- **`npm ci`** instead of `npm install` for reproducible builds in CI.
- **`npx tsc -b`** for type-check instead of `npm run build` because we only need type-checking here, not a full Vite build (the Cypress workflow already does `npm run build`).
- **No Supabase env vars needed** — unit tests mock Supabase entirely; only Cypress needs real/mock env vars.
- **Cache: 'npm'** on setup-node caches the npm global cache for faster installs.
