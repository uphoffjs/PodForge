**1. Create `.github/workflows/mutation.yml`:**

```yaml
name: Mutation Testing

on:
  pull_request:
    branches: [main]

jobs:
  stryker:
    name: Stryker Mutation Testing
    runs-on: ubuntu-24.04
    timeout-minutes: 30

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run mutation testing
        run: npm run test:mutation
```

**Design decisions:**
- **PR-only trigger** (`pull_request` only, no `push`) — mutation testing is slow (several minutes) and expensive. Running on every push to main is wasteful since those commits already passed the PR gate.
- **30-minute timeout** — Stryker can be slow on large projects. The default GitHub Actions timeout is 360 minutes, which is too generous. 30 minutes is enough for this project size but prevents runaway jobs.
- **No Supabase env vars needed** — Stryker runs Vitest which mocks all Supabase calls.
- **Uses npm run test:mutation** — the script added in Plan 01 which runs `npx stryker run`.
- **The break threshold in stryker.config.mjs** controls the CI gate — if the mutation score drops below `break`, Stryker exits with non-zero status code, failing the workflow.

**2. Update `stryker.config.mjs` thresholds:**

Update the thresholds to reflect the improved mutation score:

```javascript
thresholds: {
  high: 90,
  low: 80,
  break: 80,
},
```

Keep `break` at 80 as the CI gate (the global CLAUDE.md rule). Set `high` to 90 to flag files with excellent coverage in the report. The actual project score should be well above 80% with the new test files.

**3. Add the `html` reporter to stryker.config.mjs for CI artifacts (optional but useful):**

Add `'html'` to the reporters array so that a mutation report HTML file is generated. This can be uploaded as a CI artifact for inspection:

```javascript
reporters: ['clear-text', 'progress', 'html'],
```

Then add an artifact upload step to the workflow:

```yaml
      - name: Upload mutation report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: mutation-report
          path: reports/mutation/
          if-no-files-found: ignore
```
