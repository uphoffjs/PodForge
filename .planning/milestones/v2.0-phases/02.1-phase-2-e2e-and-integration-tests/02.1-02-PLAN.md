This spec needs a setup where there are 2+ rounds. The current round (highest round_number) is displayed via RoundDisplay. Previous rounds (lower round_numbers) are displayed via PreviousRounds in collapsible sections.

Define mock data inline:
- Round 1: { id: 'round-1', event_id: 'test-uuid', round_number: 1, created_at: '...' }
- Round 2: { id: 'round-2', event_id: 'test-uuid', round_number: 2, created_at: '...' }
- Pods for round 1: 2 pods with players (use inline fixture or reference pods.json)
- Pods for round 2: similar structure

Create `setupPageWithRounds(options)` helper that accepts rounds array, current round, and per-round pod data:
1. Block Realtime WebSocket
2. Mock event GET
3. Mock players GET (at least 4 active players)
4. Mock rounds GET — return rounds array. For the `limit=1` variant (useCurrentRound), use route handler function to differentiate by URL and return single object with `application/vnd.pgrst.object+json` header.
5. Mock pods GET — use route handler function to inspect `round_id` filter and return appropriate pods for each round. The PreviousRounds component lazy-loads pods when a section is expanded (usePods with roundId). The intercept must handle `round_id=eq.{id}` pattern. Also handle useAllRoundsPods which uses `round_id=in.(...)` pattern.
6. Visit with onBeforeLoad setting localStorage player key (admin not required for viewing rounds/pods)

Test cases:
1. "shows Previous Rounds section when multiple rounds exist" — setup with 2 rounds, assert previous-rounds container is visible
2. "does not show Previous Rounds when only one round exists" — setup with 1 round only, assert previous-rounds does NOT exist
3. "displays previous rounds in order (most recent first among previous)" — setup with 3 rounds (round 1, 2, 3). Current round is 3 (shown by RoundDisplay). Previous rounds section shows round 2, then round 1. Assert previous-round-2 appears before previous-round-1 in DOM.
4. "previous round section is collapsed by default" — assert previous-round-1 section is visible but pod cards within it are NOT visible (content collapsed)
5. "expands previous round on toggle click" —
   - Click previous-round-toggle-1
   - Mock pods GET for round-1 if not already mocked
   - Assert pod cards become visible within the expanded section (pod-card-1, pod-card-2, etc.)
   - Assert player names are displayed within pods
6. "collapses previous round on second toggle click" —
   - Click toggle to expand -> pods visible
   - Click toggle again -> pods hidden (collapsed)
7. "lazy-loads pod data only when expanded" —
   - Setup with route handler that tracks calls
   - Assert NO pods request is made for round-1 initially
   - Click previous-round-toggle-1
   - Assert pods request IS made for round-1 (cy.wait('@getPodsRound1'))

Note on mock approach for lazy loading test: The PreviousRounds component calls `usePods(isExpanded ? roundId : undefined)`. When undefined, the query is disabled (enabled: !!roundId). So no pods request fires until the user clicks to expand. Set up a specific intercept alias for pods with round-1 ID and verify it's only called after the toggle click.

For pods intercept differentiation:
```js
cy.intercept('GET', '**/rest/v1/pods*', (req) => {
  if (req.url.includes('round_id=eq.round-1')) {
    req.reply({ statusCode: 200, body: podsForRound1 })
  } else if (req.url.includes('round_id=eq.round-2')) {
    req.reply({ statusCode: 200, body: podsForRound2 })
  } else if (req.url.includes('round_id=in.')) {
    // useAllRoundsPods query
    req.reply({ statusCode: 200, body: [...podsForRound1, ...podsForRound2] })
  } else {
    req.reply({ statusCode: 200, body: [] })
  }
}).as('getPods')
```

Note: To test lazy loading, set up separate named intercepts:
```js
cy.intercept('GET', '**/rest/v1/pods*round_id=eq.round-1*', { body: podsForRound1 }).as('getPodsRound1')
```
Then verify `cy.get('@getPodsRound1.all').should('have.length', 0)` before click and `cy.wait('@getPodsRound1')` after click.

Actually, a simpler approach: Use `cy.intercept` with a callback that matches on round_id and track whether the intercept was called. But Cypress doesn't easily support negative assertions on intercepts pre-trigger. Instead:
- Set up the intercept before visiting
- After page load and before expanding, use `cy.get('@getPodsRound1.all').should('have.length', 0)` to assert no calls were made
- Click toggle to expand
- `cy.wait('@getPodsRound1')` succeeds
