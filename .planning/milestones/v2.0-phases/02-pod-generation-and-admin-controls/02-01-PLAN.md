  <action>
Create a new Supabase migration file `supabase/migrations/00002_rounds_pods_admin.sql` with:

**Tables:**

1. `rounds` table:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `event_id UUID NOT NULL REFERENCES events(id) ON DELETE CASCADE`
   - `round_number INTEGER NOT NULL`
   - `created_at TIMESTAMPTZ DEFAULT now()`
   - `UNIQUE(event_id, round_number)`

2. `pods` table:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `round_id UUID NOT NULL REFERENCES rounds(id) ON DELETE CASCADE`
   - `pod_number INTEGER NOT NULL`
   - `is_bye BOOLEAN DEFAULT false`

3. `pod_players` table:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `pod_id UUID NOT NULL REFERENCES pods(id) ON DELETE CASCADE`
   - `player_id UUID NOT NULL REFERENCES players(id) ON DELETE CASCADE`
   - `seat_number INTEGER` (null for bye pod players)

**RLS:**
- Enable RLS on all three tables
- Public read (SELECT) on all three tables (needed for Realtime delivery)
- No direct INSERT/UPDATE/DELETE -- all writes via SECURITY DEFINER RPC functions

**Realtime:**
- Add rounds, pods, pod_players to `supabase_realtime` publication

**RPC Functions (all SECURITY DEFINER with `SET search_path = public, extensions`):**

1. `generate_round(p_event_id UUID, p_passphrase TEXT, p_pod_assignments JSONB)`:
   - Validates passphrase against events.passphrase_hash using `crypt()`.
   - Validates event status is 'active' (raise exception if ended).
   - Counts active players; raises exception 'Fewer than 4 active players' if count < 4.
   - Determines next round_number as `COALESCE(MAX(round_number), 0) + 1` from rounds where event_id matches.
   - Inserts round row.
   - Iterates p_pod_assignments JSONB array. Each element: `{ "pod_number": int, "is_bye": bool, "players": [{ "player_id": uuid, "seat_number": int|null }] }`.
   - Inserts pod row per assignment, then pod_players rows.
   - Returns the new round_number as INTEGER.

2. `remove_player(p_event_id UUID, p_passphrase TEXT, p_player_id UUID)`:
   - Validates passphrase.
   - Updates player status to 'dropped' WHERE id = p_player_id AND event_id = p_event_id AND status = 'active'.
   - Returns VOID.

3. `reactivate_player(p_event_id UUID, p_passphrase TEXT, p_player_id UUID)`:
   - Validates passphrase.
   - Updates player status to 'active' WHERE id = p_player_id AND event_id = p_event_id AND status = 'dropped'.
   - Returns VOID.

4. `end_event(p_event_id UUID, p_passphrase TEXT)`:
   - Validates passphrase.
   - Updates events SET status = 'ended' WHERE id = p_event_id AND status = 'active'.
   - Returns VOID.

**Important:** The passphrase validation pattern must match the existing `validate_passphrase` function: `crypt(p_passphrase, v_hash) = v_hash` where v_hash is fetched from events.passphrase_hash. Raise EXCEPTION on mismatch (not return false) so the client gets an error.

Also add an RLS UPDATE policy on the players table for admin operations: `CREATE POLICY "Admin can update player status via RPC" ON players FOR UPDATE TO anon USING (true) WITH CHECK (status IN ('active', 'dropped'));` -- this replaces the existing policy that only allows updating to 'dropped'. Drop the old policy first.
