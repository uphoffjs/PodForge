Create migration file `supabase/migrations/00004_timer_system.sql` with:

**round_timers table:**
```sql
CREATE TABLE round_timers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  round_id UUID NOT NULL REFERENCES rounds(id) ON DELETE CASCADE UNIQUE,
  event_id UUID NOT NULL REFERENCES events(id) ON DELETE CASCADE,
  duration_minutes INTEGER NOT NULL,
  status TEXT NOT NULL DEFAULT 'running' CHECK (status IN ('running', 'paused', 'cancelled')),
  started_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  remaining_seconds INTEGER,  -- NULL when running (calculate from started_at + duration); set on pause
  paused_at TIMESTAMPTZ,      -- set when paused, cleared on resume
  expires_at TIMESTAMPTZ NOT NULL,  -- server-computed: started_at + duration (adjusted on resume/extend)
  created_at TIMESTAMPTZ DEFAULT now()
);
```

Design rationale for server-authoritative countdown:
- `expires_at` is the source of truth for when the timer reaches zero. Clients compute `remaining = expires_at - now()`.
- On pause: `remaining_seconds = EXTRACT(EPOCH FROM (expires_at - now()))`, `paused_at = now()`, status = 'paused'
- On resume: `expires_at = now() + remaining_seconds * interval '1 second'`, clear `remaining_seconds` and `paused_at`, status = 'running'
- On extend: `expires_at = expires_at + interval '5 minutes'` (works whether running or paused since paused stores remaining_seconds)
- On cancel: status = 'cancelled'
- `event_id` is denormalized for efficient Realtime filtering (avoids joining through rounds)

```sql
-- Create timer if duration specified
IF p_timer_duration_minutes IS NOT NULL AND p_timer_duration_minutes > 0 THEN
  -- Cancel any existing running/paused timer for this event
  UPDATE round_timers SET status = 'cancelled'
  WHERE event_id = p_event_id AND status IN ('running', 'paused');

  INSERT INTO round_timers (round_id, event_id, duration_minutes, expires_at)
  VALUES (v_round_id, p_event_id, p_timer_duration_minutes, now() + (p_timer_duration_minutes || ' minutes')::INTERVAL);
END IF;
```

**4 new RPC functions** (all SECURITY DEFINER, search_path = public, extensions):

1. `pause_timer(p_event_id UUID, p_passphrase TEXT)` — validates passphrase, finds running timer for event, sets remaining_seconds from (expires_at - now()), sets paused_at = now(), sets status = 'paused'. Raises exception if no running timer.

2. `resume_timer(p_event_id UUID, p_passphrase TEXT)` — validates passphrase, finds paused timer, sets expires_at = now() + (remaining_seconds * interval '1 second'), clears remaining_seconds and paused_at, sets status = 'running'. Raises exception if no paused timer.

3. `extend_timer(p_event_id UUID, p_passphrase TEXT, p_minutes INTEGER DEFAULT 5)` — validates passphrase, finds running or paused timer. If running: expires_at += p_minutes minutes. If paused: remaining_seconds += p_minutes * 60. Raises exception if no active timer.

4. `cancel_timer(p_event_id UUID, p_passphrase TEXT)` — validates passphrase, finds running or paused timer, sets status = 'cancelled'. Raises exception if no active timer.

All 4 RPCs follow the passphrase validation pattern from existing RPCs: SELECT passphrase_hash FROM events, then crypt comparison with RAISE EXCEPTION.

**2. Create `src/hooks/useTimer.ts`:**
Query hook that fetches the current active timer for an event. Pattern follows useCurrentRound.
```typescript
import { useQuery } from '@tanstack/react-query'
import { supabase } from '@/lib/supabase'
import type { RoundTimer } from '@/types/database'

export function useTimer(eventId: string) {
  return useQuery<RoundTimer | null>({
    queryKey: ['timer', eventId],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('round_timers')
        .select('*')
        .eq('event_id', eventId)
        .in('status', ['running', 'paused'])
        .order('created_at', { ascending: false })
        .limit(1)
        .maybeSingle()

      if (error) throw error
      return (data as RoundTimer) ?? null
    },
    enabled: !!eventId,
    staleTime: 5_000,  // Lower staleTime than rounds (5s) since timer state changes frequently
  })
}
```
