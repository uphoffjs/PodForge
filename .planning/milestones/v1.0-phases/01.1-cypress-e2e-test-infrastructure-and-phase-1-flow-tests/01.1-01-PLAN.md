  <action>
    1. Install dev dependencies:
       ```bash
       npm install --save-dev cypress eslint-plugin-cypress cypress-visual-regression
       ```

    2. Create `cypress.config.js` (ESM — project has "type": "module"):
       - Use `defineConfig` from cypress
       - Import `configureVisualRegression` from `cypress-visual-regression`
       - Set `e2e.baseUrl` to `http://localhost:5173`
       - Set `e2e.specPattern` to `cypress/e2e/**/*.cy.js`
       - Set `e2e.supportFile` to `cypress/support/e2e.js`
       - Set `video: true`, `screenshotOnRunFailure: true`
       - Set `screenshotsFolder: 'cypress/snapshots/actual'`
       - Set `retries: 0` (per user decision: no retries, tests must be deterministic)
       - Set env vars for visual regression:
         - `visualRegressionType: 'regression'`
         - `visualRegressionBaseDirectory: 'cypress/snapshots/base'`
         - `visualRegressionDiffDirectory: 'cypress/snapshots/diff'`
         - `visualRegressionGenerateDiff: 'fail'`
         - `SUPABASE_URL` and `SUPABASE_ANON_KEY` as empty placeholders
       - In `setupNodeEvents`, call `configureVisualRegression(on)`

    3. Create `cypress/support/e2e.js`:
       - Import and call `addCompareSnapshotCommand` from `cypress-visual-regression/dist/command.js`
       - Import `./commands.js`
       - Add `beforeEach` to clear localStorage and sessionStorage (pitfall #2 from research: state leaking between tests)
       - Add `Cypress.on('uncaught:exception')` handler to suppress Supabase Realtime WebSocket errors in mocked tests (pitfall #1)

    4. Create `cypress/support/commands.js` with these custom commands:
       - `cy.getByTestId(testId)` — returns `cy.get(\`[data-testid="${testId}"]\`)`
       - `cy.mockEventPage(eventData, playersData)` — intercepts GET events, GET players, blocks Realtime WebSocket, visits `/event/{id}`, waits on aliases. Default event: `{ id: 'test-uuid', name: 'Test Event', status: 'active', created_at: '2026-01-01T00:00:00Z' }`. Default players: empty array.
       - `cy.createRealEvent(name, passphrase)` — cy.request POST to `${Cypress.env('SUPABASE_URL')}/rest/v1/rpc/create_event` with apikey header, returns response
       - `cy.mockLandingPage()` — intercepts any background requests the landing page might make, visits `/`

    5. Create fixture files:
       - `cypress/fixtures/event.json`: Standard event response shape `{ id: 'test-uuid', name: 'Friday Night Commander', status: 'active', created_at: '2026-01-01T00:00:00Z' }`
       - `cypress/fixtures/players.json`: Array of 3 players (Alice active, Bob active, Charlie dropped) with event_id: 'test-uuid'
       - `cypress/fixtures/error-duplicate.json`: PostgreSQL 23505 error shape `{ code: '23505', details: 'Key (event_id, name)=(test-uuid, Alice) already exists.', hint: null, message: 'duplicate key value violates unique constraint "players_event_id_name_key"' }`

    6. Update `eslint.config.js`:
       - Add `import pluginCypress from 'eslint-plugin-cypress'` at top
       - Add a new config object to the array for `files: ['cypress/**/*.js']` with `...pluginCypress.configs.recommended`
       - Keep all existing TypeScript/React config unchanged

    7. Update `.gitignore`:
       - Add `cypress/snapshots/actual/` (current run screenshots)
       - Add `cypress/snapshots/diff/` (diff images on failure)
       - Add `cypress/videos/` (test run videos)
       - Do NOT add `cypress/snapshots/base/` (baselines must be committed)

    8. Create `.github/workflows/cypress.yml`:
       - Trigger on push to main and pull_request to main
       - Single job `cypress` on `ubuntu-24.04`
       - Steps: checkout@v4, then `cypress-io/github-action@v6` with:
         - `build: npm run build`
         - `start: npm run dev -- --host` (--host required for Linux CI, pitfall #4)
         - `wait-on: 'http://127.0.0.1:5173'` (use 127.0.0.1 not localhost, pitfall #7)
         - `wait-on-timeout: 60`
       - Env vars: `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` from secrets
       - On failure: upload-artifact@v4 for screenshots (cypress/snapshots/actual), diffs (cypress/snapshots/diff), and videos (cypress/videos) — each with `if-no-files-found: ignore`

    9. Add npm scripts to package.json:
       - `"cy:open": "cypress open"` — local interactive mode
       - `"cy:run": "cypress run"` — headless run
       - `"cy:base": "CYPRESS_visualRegressionType=base cypress run --spec cypress/e2e/visual-regression.cy.js"` — generate visual baselines
  </action>
