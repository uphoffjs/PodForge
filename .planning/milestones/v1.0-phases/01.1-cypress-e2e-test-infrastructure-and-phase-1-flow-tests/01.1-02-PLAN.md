  <action>
    Create two spec files covering the event creation and player join user flows. Use `cy.intercept()` to mock all Supabase PostgREST endpoints. Use `cy.getByTestId()` for all selectors. Never use `cy.wait(ms)` with arbitrary timeouts — use `cy.wait('@alias')` for network and `.should()` for DOM assertions.

    **cypress/e2e/event-creation.cy.js:**

    `describe('Event Creation')`:

    Happy path tests:
    - `it('opens create event modal when button is clicked')` — Visit `/`, click `landing-create-event-btn`, assert `create-event-modal` is visible
    - `it('creates an event and redirects to event page')` — Mock `POST **/rest/v1/rpc/create_event` to return a UUID. Fill `create-event-name-input` with event name, fill `create-event-passphrase-input` with passphrase, click `create-event-submit-btn`. Wait for `@createEvent`. Assert URL changes to `/event/{uuid}`. Mock the event page GET requests so the redirect succeeds.
    - `it('closes modal when close button is clicked')` — Open modal, click `create-event-close-btn`, assert modal is not visible
    - `it('closes modal when clicking outside')` — Open modal, click on the overlay (outside the form card), assert modal is not visible

    Error/edge case tests:
    - `it('shows validation error for empty event name')` — Open modal, leave name empty, click submit, assert form does not submit (modal stays open). Check for any validation feedback.
    - `it('shows validation error for empty passphrase')` — Open modal, fill name but leave passphrase empty, click submit, assert form does not submit
    - `it('handles server error on event creation')` — Mock create_event RPC to return 500. Fill and submit form. Assert error is displayed to user.

    **cypress/e2e/player-join.cy.js:**

    `describe('Player Join')`:

    Happy path tests:
    - `it('shows join form on event page for new visitor')` — Use `cy.mockEventPage()` with no players. Assert `join-form` is visible, `join-name-input` is visible.
    - `it('joins an event by entering name and submitting')` — Mock event page. Mock `POST **/rest/v1/players*` to return 201 with a player object. Type name into `join-name-input`, click `join-submit-btn`. Wait for `@joinPlayer`. Assert join form disappears and player appears in `player-list`. Set localStorage to simulate player identity so the UI transitions correctly.
    - `it('shows player list with existing players')` — Use `cy.mockEventPage()` with fixture players array (load from fixtures/players.json). Assert `player-list` contains expected player names.
    - `it('shows skip link for admin access')` — Mock event page. Assert `join-skip-btn` is visible.

    Edge case tests:
    - `it('shows empty state when no players exist')` — Mock event page with empty players. Assert `player-list-empty` is visible.
    - `it('trims whitespace from player name')` — Mock event page. Type "  Alice  " into name input, submit. Assert the trimmed name is sent in the request body (verify via `cy.wait('@joinPlayer').its('request.body')` or equivalent).

    **Rules for both specs:**
    - Each test starts fresh (beforeEach clears storage — handled by support file)
    - Use `cy.intercept()` for ALL Supabase calls — no real network
    - Block Realtime WebSocket: `cy.intercept('GET', '**/realtime/**', { statusCode: 200, body: {} })`
    - All selectors use `cy.getByTestId()`, never CSS classes or tag names
    - No `cy.wait(ms)` — only `cy.wait('@alias')` or `.should()` assertions
