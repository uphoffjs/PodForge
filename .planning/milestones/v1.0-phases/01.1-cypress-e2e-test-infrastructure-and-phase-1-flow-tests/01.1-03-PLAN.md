    Create the visual regression spec file with screenshot comparison at 3 breakpoints for both the landing page and event page.

    **cypress/e2e/visual-regression.cy.js:**

    Define breakpoints array:
    ```javascript
    const breakpoints = [
      { name: 'mobile', width: 375, height: 812 },
      { name: 'tablet', width: 768, height: 1024 },
      { name: 'desktop', width: 1280, height: 800 }
    ]
    ```

    `describe('Visual Regression - Landing Page')`:
    - For each breakpoint: set viewport, visit `/`, wait for fonts to load (`cy.document().its('fonts.status').should('equal', 'loaded')`), then call `cy.compareSnapshot(\`landing-${name}\`, 0.05)` (5% threshold per research recommendation)

    `describe('Visual Regression - Event Page (with players)')`:
    - For each breakpoint: set viewport, call `cy.mockEventPage()` with a named event and a mix of active + dropped players (use fixture data pattern: 2 active, 1 dropped). Wait for fonts. Call `cy.compareSnapshot(\`event-page-${name}\`, 0.05)`.

    `describe('Visual Regression - Event Page (empty state)')`:
    - For each breakpoint: set viewport, call `cy.mockEventPage()` with no players. Wait for fonts. Call `cy.compareSnapshot(\`event-empty-${name}\`, 0.05)`.

    `describe('Visual Regression - Event Page (join form)')`:
    - For each breakpoint: set viewport, call `cy.mockEventPage()` with players BUT no localStorage identity (so join form is shown). Wait for fonts. Call `cy.compareSnapshot(\`event-join-form-${name}\`, 0.05)`.

    **Font loading wait is critical** (pitfall #3 from research): The app uses Google Fonts CDN with `font-display: swap`. Without waiting, screenshots capture fallback fonts and produce false diffs.

    After writing the spec, generate baselines:
    ```bash
    CYPRESS_visualRegressionType=base npx cypress run --spec cypress/e2e/visual-regression.cy.js
    ```

    This creates baseline images in `cypress/snapshots/base/`. These must be committed to git so CI regression checks work (pitfall #6 from research).
