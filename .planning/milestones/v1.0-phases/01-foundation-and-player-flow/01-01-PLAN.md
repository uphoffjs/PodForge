<task type="auto">
  <name>Task 2: Create Supabase database migration with schema, RLS, and RPC functions</name>
  <files>
    supabase/migrations/00001_initial_schema.sql
  </files>
  <action>
    Create the SQL migration at `supabase/migrations/00001_initial_schema.sql`. This is the complete database foundation. Follow the exact schema from RESEARCH.md but verify every detail:

    **Enable pgcrypto:**
    ```sql
    CREATE EXTENSION IF NOT EXISTS pgcrypto WITH SCHEMA extensions;
    ```

    **Events table:**
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `name` TEXT NOT NULL with CHECK (length 1-100)
    - `passphrase_hash` TEXT NOT NULL
    - `status` TEXT NOT NULL DEFAULT 'active' CHECK (IN 'active', 'ended')
    - `created_at` TIMESTAMPTZ DEFAULT now()

    **Players table:**
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `event_id` UUID NOT NULL REFERENCES events(id) ON DELETE CASCADE
    - `name` TEXT NOT NULL with CHECK (length 2-20) -- per user decision: min 2, max 20
    - `status` TEXT NOT NULL DEFAULT 'active' CHECK (IN 'active', 'dropped')
    - `created_at` TIMESTAMPTZ DEFAULT now()
    - UNIQUE(event_id, name) -- prevents duplicate names per event

    **RLS (CRITICAL -- not retrofittable):**
    - Enable RLS on BOTH tables immediately after creation
    - Column-level security: `REVOKE SELECT ON events FROM anon` then `GRANT SELECT (id, name, status, created_at) ON events TO anon` -- hides passphrase_hash
    - SELECT policies: `USING (true)` for anon on both tables (public read -- required for Realtime)
    - INSERT policy on players: `WITH CHECK (status = 'active')` for anon (anyone can join)
    - UPDATE policy on players: `USING (true) WITH CHECK (status = 'dropped')` for anon (self-drop via RPC)

    **Realtime publication (CRITICAL -- without this, subscriptions silently receive nothing):**
    ```sql
    ALTER PUBLICATION supabase_realtime ADD TABLE events;
    ALTER PUBLICATION supabase_realtime ADD TABLE players;
    ```

    **RPC functions (all SECURITY DEFINER with `SET search_path = public, extensions`):**

    1. `create_event(p_name TEXT, p_passphrase TEXT) RETURNS UUID` -- inserts event with hashed passphrase using `crypt(p_passphrase, gen_salt('bf'))`, returns event ID

    2. `validate_passphrase(p_event_id UUID, p_passphrase TEXT) RETURNS BOOLEAN` -- fetches hash, compares with `crypt()`, returns true/false

    3. `drop_player(p_player_id UUID) RETURNS VOID` -- updates player status to 'dropped' where status = 'active' (controlled self-drop)

    **IMPORTANT:** Use `extensions.crypt()` and `extensions.gen_salt()` OR set `search_path = public, extensions` on each function. Supabase puts pgcrypto in the `extensions` schema, not `public`. Without this, you get "function does not exist" errors.
  </action>
