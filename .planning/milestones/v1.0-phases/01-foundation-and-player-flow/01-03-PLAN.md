  <action>
    **src/hooks/useEvent.ts** -- Fetch event details:
    - `useEvent(eventId: string)` uses `useQuery` from `@tanstack/react-query`
    - Query key: `['event', eventId]`
    - Query function: `supabase.from('events').select('id, name, status, created_at').eq('id', eventId).single()`
    - **CRITICAL:** Use explicit column list, NOT `select('*')`. Column-level security revokes passphrase_hash from anon, so `*` will fail.
    - `enabled: !!eventId`
    - Return the full `useQuery` result (data, isLoading, error)

    **src/hooks/useEventPlayers.ts** -- Fetch player list:
    - `useEventPlayers(eventId: string)` uses `useQuery`
    - Query key: `['players', eventId]`
    - Query function: `supabase.from('players').select('*').eq('event_id', eventId).order('created_at', { ascending: true })`
    - Note: `*` is fine on the players table (no restricted columns)
    - `enabled: !!eventId`
    - `staleTime: 30_000` (Realtime handles freshness in Plan 04)

    **src/hooks/useJoinEvent.ts** -- Join event mutation:
    - `useJoinEvent(eventId: string)` uses `useMutation`
    - `mutationFn` accepts `name: string` and calls `supabase.from('players').insert({ event_id: eventId, name }).select().single()`
    - `onSuccess(data)`:
      1. Call `storePlayerId(eventId, data.id)` from `@/lib/player-identity`
      2. Call `queryClient.invalidateQueries({ queryKey: ['players', eventId] })`
      3. Show toast: "Joined the event!" via sonner
    - `onError(error)`:
      - If `error?.code === '23505'` (unique constraint violation): toast.error("That name is already taken. Try another!")
      - Otherwise: toast.error("Failed to join. Please try again.")
    - This error handling implements PLYR-05 (duplicate name prevention with friendly error)
  </action>
  <verify>
    `npx tsc --noEmit` passes.
    All three hooks export correctly.
  </verify>
  <done>
    Three data hooks created: useEvent fetches event details with explicit column list, useEventPlayers fetches player list, useJoinEvent handles joining with duplicate name error handling (Postgres 23505 -> friendly message).
  </done>
